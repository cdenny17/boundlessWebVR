About Boundless WebVR

This is an immersive web experience. Using the Three.js JavaScript 3D library to build a 3D environment and the webXR API you can bypass using a game engine like Unity and create Virtual Reality experiences and make them readily available to users on many devices. For this project I focused on the HTC Vive so the instructions below will help you connect a Vive to a web-based virtual reality environment that runs in a browser.

This Project was Produced for the WSU Summer Mini Grant by Christian Denny Using the Immersive Web Group and W3C webXR API

Getting Started

The webXR API is a new work in process. At the time of this release getting it to work requires use of Google Chrome with some settings changes. Follow this link to chrome://flags to make settings changes to enable VR devices. Enable these settings: WebVR | WebXR Device API | WebXR Hit Test | WebXR Plane Detection | Orientation sensors for XR | Oculus hardware support | OpenVR hardware support | Windows Mixed Reality support. Disable this setting: XR device sandboxing.

You also have to have a VR device. This particular project has been created and tested with the HTC Vive. The purpose of the webXR API is to integrate all VR and AR hardware with web base applications so in the future use of the Oculus or Google Cardboard is a possibility

Download and run Steam and Steam VR. Steam acts as a bridge between the browser and the Vive. Even though this project bipasses the need to upload the VR content to Steam, it is impossible to eliminate Steam altogether at this time. Follow Steam's instructions on downloading and setting up Steam VR using HTC Vive.
